Media Optimization and Copyright
 

Week 2

Different types of media files play different roles on the web today. Images (JPEG, PNG, GIF, etc.) are commonly used for visuals such as product photos on some e-commercial sites, icons for navigation menus, and backgrounds. Videos (MP4, etc.) offer dynamic storytelling and tutorials. Source code files (HTML, CSS, JavaScript, etc.) form the backbone of webpages, and enable structure, styling, and interactivity. Project files like PDFs and ZIP archives allow users to download bundled resources or documents such as website templates, academic papers, etc.

Images used on the web should be optimized to ensure fast load times without sacrificing quality. Cropping removes unnecessary parts of an image, focusing attention on key elements. Resizing ensures the image fits its intended display size and avoids scaling issues. Compression reduces file size, with lossy compression (e.g., JPEG) and offers smaller files at the cost of some quality and lossless compression (e.g., PNG) preserving original quality but with larger file sizes. Tools like Photoshop can help with compression. I used Preview on Mac and online converter to compress this image. Also, WebP is a good modern format for compressed images with smaller sizes than JPEG/PNG.

Optimizing videos for the web involves capturing in a suitable resolution, cutting unnecessary sections, resizing to meet display requirements, and compressing to reduce file size. HD resolution (1280x720px) strikes a good balance between quality and size. Tools like Adobe Premiere Pro or some online converters allow resizing and compression. The MP4 format (H.264 codec) is widely supported for web use. I use free convertor online by Adobe to compress this video.

Hosting videos on platforms like YouTube or Vimeo offers additional benefits, including automated compression, global accessibility, and built-in playback controls. These platforms also handle video compatibility across devices, reducing the need for developers to encode videos for different browsers. However, embedding external videos can come with drawbacks like ads or branding that may disrupt user experience. Self-hosting small, optimized videos is a better option for cleaner integration.
